2. Técnicas para Datos Cuantitativos (Números)
Cuando tienes números de encuestas, conteos y mediciones, estas técnicas te ayudan a interpretarlos.
📋

Tabla de Frecuencias

La técnica más básica y poderosa

Cuenta cuántas veces aparece cada respuesta. Te dice qué opciones son más comunes y cuáles son minoritarias.

  1. 1 Lista todas las respuestas posibles en una columna
  2. 2 Cuenta cuántas veces aparece cada una
  3. 3 Calcula el porcentaje de cada respuesta
  4. 4 Ordena de mayor a menor frecuencia

Pregunta de encuesta: "¿Qué funcionalidad quiere en el sistema?"

📗 Excel — Tabla de Frecuencias
RespuestaConteoFrecuencia %
Registro de ventas1890%
Control de inventario1680%
Factura electrónica1470%
Control de proveedores840%
App para clientes420%
Fórmula: =CONTAR.SI(B:B,"Registro de ventas")

💡 Conclusión: El 90% quiere registro de ventas → Es un requisito PRIORITARIO del sistema.

📐

Estadísticas Descriptivas

Promedio, mínimo, máximo, moda

Resumen numérico básico de tus datos. Sin necesidad de estadística avanzada, estas métricas te dan una visión rápida de la realidad.

  1. 1 Ingresa los datos en Excel (una columna por variable)
  2. 2 Calcula promedio (PROMEDIO), máximo (MAX), mínimo (MIN)
  3. 3 Calcula la moda (MODA) para saber el valor más frecuente
  4. 4 Interpreta qué significa cada número para el proyecto

Variable: "¿Cuántas ventas diarias hace su negocio?"

📗 Excel — Estadísticas descriptivas
MétricaValorFórmula Excel
Promedio diario47 ventas=PROMEDIO(B2:B20)
Mínimo registrado22 ventas=MIN(B2:B20)
Máximo registrado89 ventas=MAX(B2:B20)
Día más comúnSábado=MODA(C2:C20)

💡 Conclusión: El sistema debe soportar mínimo 90 transacciones/día (con margen de crecimiento). La carga pico es los sábados → considerar rendimiento.

Gráficas en Excel

Visualizar para entender mejor

Una gráfica bien hecha comunica en segundos lo que una tabla tarda minutos en explicar. En Excel puedes crear gráficas sin conocimientos avanzados.

  1. 1 Selecciona los datos de tu tabla de frecuencias
  2. 2 Insertar → Gráfico → elige el tipo apropiado
  3. 3 Agrega título, etiquetas y leyenda
  4. 4 Exporta como imagen para el informe

¿Qué gráfica usar según los datos?

Matriz de Priorización

Urgente vs. Importante

No todo lo que los usuarios piden tiene la misma importancia. Esta técnica ayuda a ordenar los requisitos según su impacto y urgencia.

  1. 1 Lista todos los requisitos identificados
  2. 2 Califica cada uno: Importancia (1-5) y Urgencia (1-5)
  3. 3 Multiplica: Prioridad = Importancia × Urgencia
  4. 4 Ordena de mayor a menor prioridad

3. Paso a Paso en Excel para Analizar tu Encuesta
Guía práctica para organizar los datos de tu encuesta en Excel desde cero.
01

Crear la hoja de datos

Primera fila: nombres de las preguntas como encabezados. Cada fila siguiente: respuestas de un participante.

Columna A: ID | B: Pregunta1 | C: Pregunta2...
02

Limpiar datos

Eliminar filas vacías, corregir inconsistencias (Sí / SI / si → unificar). Usar Buscar y Reemplazar (Ctrl+H).

Datos → Quitar duplicados
03

Contar respuestas

Para cada opción de respuesta, usa CONTAR.SI para saber cuántas veces aparece.

=CONTAR.SI(B:B,"Sí")
04

Calcular porcentajes

Divide el conteo entre el total de respuestas y multiplica por 100. Formatea como porcentaje.

=C2/SUMA(C:C)*100
05

Crear tabla resumen

Nueva hoja con: Respuesta | Conteo | % | Interpretación. Esta será la base de tu informe.

Insertar → Tabla (Ctrl+T)
06

Insertar gráfica

Selecciona la tabla resumen. Insertar → Gráfico de barras. Agrega título descriptivo y etiquetas de datos.

Insertar → Gráfico recomendado

4. Técnicas para Datos Cualitativos (Textos y Opiniones)
Los datos de entrevistas y observaciones no son números — pero también tienen patrones que debes descubrir.

🏷️ Categorización y Etiquetado

Lee todas las respuestas de las entrevistas y asigna etiquetas (categorías) a cada idea expresada. Luego agrupa las etiquetas similares.

Ejemplo: Respuesta: "A veces no sé cuánto llevo vendido en el día" → Etiquetas:

Control de ventas Visibilidad en tiempo real Reportes diarios

🔢 Frecuencia de Temas

Cuenta cuántas veces aparece cada categoría en todas las entrevistas. Los temas más frecuentes son los más importantes para el sistema.

Ejemplo: Tema "inventario" mencionado por:

Dueño ✓ Empleado ✓ Esposa ✓

→ Mencionado por 3/3 usuarios = requisito crítico

🗂️ Agrupación por Módulos

Clasifica los hallazgos según el módulo del sistema al que corresponden. Esto te da la estructura inicial del software.

📦 Inventario 💰 Ventas
🚚 Proveedores 👥 Usuarios

💬 Citas Textuales Clave

Selecciona frases exactas de las entrevistas que expresan claramente una necesidad. Úsalas en el informe como evidencia.

"No sé cuándo se va a acabar un producto hasta que ya no hay" — Don Carlos, dueño

→ Requisito: Alertas de stock mínimo

🔁 Identificar Contradicciones

A veces diferentes usuarios quieren cosas opuestas. Documentarlas y discutirlas con el cliente es parte del análisis.

Ejemplo de contradicción:

Dueño: "Solo yo debo ver las ventas totales" vs. Empleado: "Necesito ver mi rendimiento del día"

→ Solución: sistema de roles y permisos

📝 Lluvia de Requisitos

Después de analizar todo, el equipo hace una lista de todas las necesidades identificadas. Sin filtrar — primero cantidad, luego calidad.

Herramienta: Post-its en pared o tabla en Excel/Word con columnas: ID | Descripción | Fuente | Tipo.

5. Errores Comunes al Analizar Datos
Conocer los errores más frecuentes te ayuda a evitarlos en tu propio análisis.
😴

Copiar sin analizar

Pegar las respuestas textuales en el informe sin transformarlas en conclusiones ni requisitos.

✅ Siempre cierra con: "Esto significa que el sistema debe..."
🔊

El usuario más hablador dicta todo

Quien más habla en la entrevista no siempre representa la mayoría. La frecuencia de menciones vale más que el énfasis verbal.

✅ Usa frecuencias y encuestas para validar opiniones individuales
📦

Confundir deseos con necesidades

"Quisiera una app para celular" ≠ "El negocio necesita registrar ventas". Los deseos son opcionales, las necesidades no.

✅ Pregunta: ¿el negocio puede operar sin esto? Si la respuesta es sí, es deseo
📉

Ignorar los datos que no encajan

Descartar respuestas que contradicen nuestra idea previa del sistema. Todo dato revela algo importante.

✅ Documenta todo, incluso las contradicciones — son oportunidades de diseño
🚀

Saltar directo a la solución

Empezar a diseñar pantallas antes de terminar el análisis. El análisis incompleto genera requisitos incompletos.

✅ Cierra el análisis con un documento de hallazgos ANTES de diseñar
📋

No documentar las fuentes

Escribir "el usuario dijo X" sin registrar quién, cuándo y en qué instrumento. Los datos sin fuente no son verificables.

✅ Siempre registra: quién dijo qué, cuándo y en qué instrumento
